# 1. Purpose

The Requirements Traceability Matrix (RTM) establishes traceability between business requirements, functional requirements, use cases, architecture, implementation, and testing.

# Coverage Rules

Every business requirement shall trace to:

- One or more functional requirements
- One or more use cases
- One architectural component
- One or more verification tests

No requirement shall remain untraced.

---

# Verification

The RTM shall be updated for:

- New requirements
- Requirement changes
- Feature additions
- Architectural changes
- Test additions
