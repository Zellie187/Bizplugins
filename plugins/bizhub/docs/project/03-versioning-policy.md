# Version Format

Semantic Versioning

```
MAJOR.MINOR.PATCH
```

---

# Major Version

Increment when:

- Breaking API changes
- Breaking database changes
- Major architecture changes

---

# Minor Version

Increment when:

- New features
- New modules
- Backward-compatible enhancements

---

# Patch Version

Increment when:

- Bug fixes
- Security fixes
- Documentation corrections
- Performance improvements

---

# Release Tags

Examples:

```
v1.0.0
v1.1.0
v1.1.1
```

---

# Pre-release Tags

```
v2.0.0-alpha.1

v2.0.0-beta.1

v2.0.0-rc.1
```
